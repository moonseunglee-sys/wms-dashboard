**정렬: C 작업자(가나다) → A 시간 오름차순** (7/1 검증 완료, 작업자별 연속 블록)

### 핵심 수식
```
J(입고유형) = IF(H="Y-REC-00","CUT", IF(I="반품입고","반품입고","정상입고"))
L(입고수량) = IF(J ∈ {정상입고,반품입고,정품화입고,CUT,재입고,"검사이동,업체반송"}, E, 0)
M(입고금액) = 같은 조건 × VLOOKUP(D, '일-단품정보'!D:F, 3, 0) × E   ← F열=공장도가
N(파렛트)   = IF(J ∈ {정상입고,반품입고,정품화입고,재입고,검사이동}, 1, 0)   ← CUT 제외
T(소요시간) = 같은 작업자 연속 스캔 간격(분)
              12시→13시 걸치면 -50(점심), 17시→18시 걸치면 -30(저녁)
              작업자 바뀌면 0 (첫 행 무시)
```
- J 수식이 낼 수 있는 값: CUT / 반품입고 / 정상입고 뿐
- L·N 조건의 정품화입고/재입고/"검사이동,업체반송"은 **수동 입력 전용 값**
- 이동행은 I가 비어 J=정상입고 처리됨 (QCC 반출입 모두 정상입고로 카운트)

### 수동 예외 (사용자 가이드, 7월·6월 시트에는 실사례 없음)
1. `MOVER_RFxxx` → `MOVER` 통일
2. 반품입고 다품목 입하: raw 입고유형이 재고이동(반입)으로 나오는 건 → From=Y-RET-01 대조로 찾아 J="반품입고" 수동
3. 정상입고 다품목: J=CUT인데 To팔레트에 단말기 번호 → J="정상입고" 수동
4. CUT 일부+정상입고 일부 분할 케이스 → 개별 점검 필요
